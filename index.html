Pokemon chain of responsibility (in VanillaJS)

The chain-of-responsibility pattern is a behavioral design pattern consisting of a source of command objects and a series of processing objects. Each processing object contains logic that defines the types of command objects that it can handle; the rest are passed to the next processing object in the chain. Read the wikipedia article

The purpose of this excercise is to implement this design pattern in Vanilla JavaScript creating a list of pokemon to fetch and then display the results (using PokeAPI). If some service call fails then chain breaks and no other service call is performed. (try adding number 9999, there aren't that many pokemon... yet.)
